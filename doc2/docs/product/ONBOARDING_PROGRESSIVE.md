## 2. ONBOARDING PHASES

### Phase 0: First Contact (5 minutes)

**Goal:** Understand what CHEÂ·NU is

```yaml
Phase 0 - First Contact:
  Duration: 5 minutes
  
  What User Sees:
```

### Phase 1: First Sphere (Day 1-3)

**Goal:** Master one sphere

```yaml
Phase 1 - First Sphere:
  Duration: 1-3 days
  Default Sphere: Personal
  
  What User Sees:
    - Personal sphere fully
    - Bureau: Dashboard + Notes + Tasks only
    - Nova always present
    - Simple thread creation
  
  What User Does:
    - Create first note
    - Create first task
    - Have first conversation with Nova
    - Complete "Personal Setup" quest
  
  Unlock Triggers:
    - 3 notes created â†’ Unlock Projects
    - 5 tasks completed â†’ Unlock Meetings
    - 10 messages with Nova â†’ Unlock Threads view
  
  Hidden:
    - Other spheres (visible but locked)
    - Data, Agents, Reports, Budget sections
    - Token details (just shows "plenty remaining")
    - Multiple agents
```

### Phase 2: Expanding Bureau (Week 1)

**Goal:** Full bureau in one sphere

```yaml
Phase 2 - Bureau Mastery:
  Duration: Week 1
  
  Unlocks (in order):
    1. Projects (after Phase 1)
    2. Meetings (after 5 tasks)
    3. Threads view (after 10 Nova messages)
    4. Data/Database (after first project)
    5. Reports (after 1 week)
    6. Budget (after Reports unlocked)
    7. Agents (after Budget understood)
  
  Nova's Role:
    - Celebrates each unlock
    - Offers mini-tutorial for new section
    - Suggests when user is ready
  
  Hidden:
    - Other spheres still locked
    - Advanced governance
    - Multi-agent operations
```

### Phase 3: Second Sphere (Week 2)

**Goal:** Understand sphere separation

```yaml
Phase 3 - Second Sphere:
  Duration: Week 2
  
  Trigger: User tries to mix contexts OR Nova suggests
  
  What Happens:
    - Nova: "I notice you're working on business tasks.
             Should I show you the Business sphere?"
    - User chooses to unlock Business
    - Business sphere unlocks with Phase 1 features
  
  Key Learning:
    - Spheres are separate contexts
    - Data doesn't automatically cross
    - Nova adapts to each sphere
  
  Hidden:
    - Remaining 6 spheres
    - Cross-sphere operations
    - Advanced token management
```

### Phase 4: Token Awareness (Week 3)

**Goal:** Understand token system

```yaml
Phase 4 - Token Awareness:
  Duration: Week 3
  
  Trigger: User approaches token limit OR asks
  
  What User Learns:
    - Tokens = intelligence energy
    - Every AI action has a cost
    - Budgets control spending
    - Encoding reduces cost
  
  Nova's Role:
    - "You've used 80% of this week's tokens"
    - "Want me to show you how encoding saves tokens?"
    - "Let's set up a budget for your Business sphere"
  
  Unlocks:
    - Budget section details
    - Token history
    - Encoding controls
```

### Phase 5: Agent Expansion (Month 1)

**Goal:** Hire first agent beyond orchestrator

```yaml
Phase 5 - Agent Expansion:
  Duration: Month 1
  
  Trigger: Complex task OR Nova suggests
  
  What User Learns:
    - Nova is system (always present)
    - Orchestrator is your hired assistant
    - Specialists have specific skills
    - Agents have scopes and costs
  
  First Hire Experience:
    - Nova: "This task needs a specialist"
    - Shows agent marketplace
    - User hires first agent
    - Agent introduces itself
    - Task executed with visible cost
```

### Phase 6: Full Access (Month 2+)

**Goal:** All features available

```yaml
Phase 6 - Full Access:
  Duration: Month 2+
  
  Everything Unlocked:
    - All 8 spheres
    - All 10 bureau sections
    - Full agent marketplace
    - Advanced governance
    - Cross-sphere operations
```
