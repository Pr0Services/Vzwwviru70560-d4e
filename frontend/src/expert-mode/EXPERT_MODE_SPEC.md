## ELIGIBILITY TO ENABLE EXPERT MODE

Expert Mode may be enabled only if **ALL** are true:

1. User has completed multiple assisted actions
2. User has accepted or rejected proposals knowingly
3. Orchestrator is activated
4. User explicitly opts in

**No automatic activation.**

## WHAT CHANGES IN EXPERT MODE

### 1. Fewer Confirmations (But Never Zero)

- Repeated actions may skip *explanatory* confirmations
- Final execution confirmation remains mandatory
- Budget, scope, and model visibility remain intact

| Normal Mode | Expert Mode |
|-------------|-------------|
| Explain ‚Üí Confirm ‚Üí Execute | Execute-ready proposal ‚Üí Confirm ‚Üí Execute |

---

### 2. Keyboard-First Interaction

Expert shortcuts become available:

| Shortcut | Action |
|----------|--------|
| `‚åò/Ctrl + Enter` | Confirm proposal |
| `‚åò/Ctrl + Shift + S` | Prepare structured version |
| `‚åò/Ctrl + K` | Open command palette |
| `Esc` | Cancel / Dismiss proposal |

**Shortcuts are discoverable but never required.**

---

### 3. Command Palette (Text-Based)

Opens with: `‚åò/Ctrl + K`

#### Examples
- "Structure selection"
- "Summarize document"
- "Prepare meeting notes"
- "Compare versions"

#### Rules
- Prepare proposals only
- Never execute directly
- Always show scope & cost before execution

---

### 4. Reduced Nova Verbosity

In Expert Mode:
- Nova explanations are shorter
- No repetition of known concepts
- Warnings still appear for:
  - Scope unlock
  - Budget thresholds
  - Sensitive actions

**Nova remains the governance layer.**

---

### 5. Persistent Preferences

Expert Mode allows:
- Default scope preference
- Default budget preset
- Default model preference (if allowed)

Preferences are:
- Explicit
- Reversible
- Visible

- New scopes require unlock
- New models require confirmation
- Budget overruns are blocked
