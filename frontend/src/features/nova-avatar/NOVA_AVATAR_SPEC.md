Nova avatar may be proposed only if **ALL** are true:

2. User has activated the orchestrator
3. User has enabled Nova voice
4. User explicitly opts in to visual presence

**No automatic activation.**
**No avatar preview before consent.**

## AVATAR APPEARANCE RULES (NON-NEGOTIABLE)

### Visual Style
- Futuristic, calm, neutral
- Semi-realistic (no uncanny valley)
- Minimal facial expression
- Slow, subtle motion only
- Neutral posture
- No gender emphasis (or soft-neutral)

### Color & Light
- Muted palette
- No bright saturation
- No aggressive contrast
- Eyes may glow subtly (status indicator only)

### Size & Placement
- Never full-screen
- Appears as a side presence
- Docked to Surface A (Dialogue)
- Can be minimized at any time

## RELATIONSHIP BETWEEN VOICE AND AVATAR

- Avatar never speaks without voice enabled
- Voice never requires avatar
- Avatar lip-sync is optional and subtle
- No idle chatter
- No filler animations

---

## AVATAR IN MEETINGS

- Avatar is hidden by default
- Can be enabled manually
- Appears as a small presence
- Never replaces human video
- Never competes visually with participants

---

## PRIVACY & TRUST GUARANTEES

- Avatar does not imply listening
- Visual presence ‚â† audio capture
- Microphone state always explicit
- Avatar disappears when voice is off

---

## PERFORMANCE CONSTRAINTS

- Avatar rendering is lightweight
- Disabled on low-performance devices
- Disabled automatically in battery-saving modes
- Never blocks core functionality

---

## ACCESSIBILITY RULES

- Full functionality without avatar
- Avatar can be permanently disabled
- No critical information conveyed visually only
- Text alternatives always available

---

## FAILURE & FALLBACK

If avatar fails to load:
- System continues in text-only mode
- No error surfaced to user
- Nova explains only if user asks
