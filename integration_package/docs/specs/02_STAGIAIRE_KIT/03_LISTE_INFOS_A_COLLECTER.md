# 03 — Liste minimale d’informations à collecter

Objectif : collecter seulement ce qui aide le futur système à décider mieux.

## A. Intention perçue
- Objectif supposé (1 phrase)
- Ambiguïtés notées (liste courte)
- Ce qui n’a pas été dit explicitement (1–3 points)

## B. Décision du système
- Route : local / sous-traité
- Confiance perçue : faible / moyenne / forte
- Moment : “fin de conversation” (toujours)

## C. Intervention externe (si applicable)
- Pourquoi l’externe a été appelé
- Ce que l’externe a apporté (utile)
- Ce qui aurait pu être évité (si clair)

## D. Réaction utilisateur (même passive)
- Satisfaction implicite / correction / reformulation / silence
- Ce qui a changé entre intention initiale et finale

## E. Potentiel d’apprentissage
- Récurrence probable : basse / moyenne / haute
- Valeur future estimée : basse / moyenne / haute
- Priorité : basse / moyenne / haute

## Ce que le stagiaire JETTE volontairement
- Évidences (cas faciles, non ambigus)
- Redondances sans surprise
- Détails contextuels non structurants (anecdotes, folklore)
- Certitudes artificielles (conclusions trop rapides)
- Cas uniques sans futur (non reproductibles)

> Si une note n’aide pas le système dans 6 mois, elle ne mérite pas d’exister.
