## 1) INTER-SPHERE KNOWLEDGE THREADS ⚡

### Purpose
> **Link related information across spheres WITHOUT merging domains or altering meaning.**

### RULE
> **Thread = CONNECTION, not interpretation.**

---

## THREAD SOURCES ⚡

| Source | Description |
|--------|-------------|
| **artifacts** | documents, notes, XR boards ⚡ |
| **events** | actions, silence markers ⚡ |
| **decisions** | time-stamped ⚡ |
| **replays** | validated ⚡ |
| **agents' factual outputs** | ⚡ |

---

## 4 THREAD TYPES ⚡

| Type | Description |
|------|-------------|
| `THREAD_TOPIC` | **shared concepts across spheres** (ex: "budget" → Business + Institution) ⚡ |
| `THREAD_ARTIFACT` | **same document referenced in multiple spheres** ⚡ |
| `THREAD_EVENT` | identical event appears in **separate contexts** ⚡ |
| `THREAD_DEPENDENCY` | **sphere B requires validated output of sphere A** ⚡ |

---

## THREAD MODEL JSON ⚡

```json
{
  "knowledge_thread": {
    "id": "uuid",
    "type": "topic|artifact|event|dependency",
    "nodes": ["sphere.event", "sphere.artifact"],
    "timestamp": 1712345678,
    "hash": "sha256",
    "integrity": "verified"
  }
}
```

### Key Field: `integrity: "verified"` ⚡
> **Cryptographic verification required**
