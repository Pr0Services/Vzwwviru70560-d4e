## Chapter 84: ShareView Governed Screen-Sharing

### Concept

ShareView is CHEÂ·NU's privacy-first approach to AI assistance during user work. Unlike invasive screen monitoring, ShareView gives users explicit control over what the AI can see and how it can help.

### Activation

Users explicitly activate ShareView when they want AI assistance with visible content. This can be:

- **Full Workspace**: AI sees entire OCW canvas
- **Selected Area**: AI sees only a designated region
- **Specific Elements**: AI sees only particular objects or documents
- **None**: AI operates only on verbal/text instructions

### Privacy Guarantees

**Explicit Boundaries**: AI sees only what user explicitly shares
**No Device Access**: ShareView never accesses device data outside the workspace
**No Background Monitoring**: AI observation is active only during explicit sessions
**Transparent State**: Clear visual indicators show what AI can see

### AI Capabilities in ShareView

When viewing shared content, AI can:

- **Interpret**: Understand what user is working on
- **Suggest**: Offer improvements, completions, transformations
- **Classify**: Propose organizational structures
- **Connect**: Identify relationships to existing DataSpaces
- **Transform**: Convert content between formats (sketch to diagram, notes to outline)

### Agent Roles in ShareView

**Spectator Mode**: Agents observe without acting, building understanding

**Advisor Mode**: Agents offer suggestions that appear as non-intrusive recommendations

**Active Mode**: Agents can make changes with user approval for each action

## Chapter 86: Agent Spectator Mode

### Governed Observation

Agents in the OCW operate in strictly governed roles, never acting without explicit permission.

### Role Definitions

**Spectator Agent**
- Observes workspace content (when ShareView active)
- Builds contextual understanding
- Does not make suggestions or modifications
- Provides foundation for other agent roles

**Advisor Agent**
- Observes workspace content
- Generates suggestions that appear as subtle recommendations
- User can accept, modify, or dismiss suggestions
- Never modifies content without user action

**Organizer Agent**
- Can propose restructuring of content
- Shows preview of proposed organization
- Executes reorganization only on user approval
- Respects user spatial intentions

**Constructor Agent**
- Builds workflows, diagrams, and structures from user sketches
- Presents constructed output alongside original
- User chooses to keep original, use construction, or blend

**Analyst Agent**
- Examines workspace for patterns, risks, insights
- Surfaces findings as non-intrusive notifications
- Provides deeper analysis on request
- Identifies connections to existing DataSpaces

### Permission Hierarchy

All agent actions require appropriate permission:

1. **View Permission**: Can agent see this content?
2. **Suggest Permission**: Can agent offer recommendations?
3. **Modify Permission**: Can agent make changes (with approval)?
4. **Auto-Act Permission**: Can agent make changes without per-action approval?

Default is view-only; each additional level requires explicit user grant.

## Chapter 96: OCW Glossary

**Operational Cognitive Workspace (OCW)**: CHEÂ·NU's primary user interface, a unified intelligent workspace.

**ShareView**: Privacy-respecting mode allowing AI to observe user-defined workspace regions.

**Canvas Layer**: The visual surface where users create and interact with content.

**Spectator Agent**: AI role that observes without acting.

**Advisor Agent**: AI role that offers suggestions without modifying content.

**Organizer Agent**: AI role that proposes restructuring of content.

**Constructor Agent**: AI role that builds structured outputs from user input.

**Analyst Agent**: AI role that identifies patterns, risks, and insights.

**Workspace Transformation**: Conversion of workspace between different modes (document, board, timeline, etc.).

**Operational Cockpit**: Dashboard mode showing key indicators and status across spheres and domains.

**Identity Adaptation**: OCW reconfiguration based on active user identity.

**XR Bridge**: System enabling transition between 2D workspace and 3D XR environment.
